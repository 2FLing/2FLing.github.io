GIT

What is GIT?

Git is a distributed version management system written by Linus, the creator of Linux. GitHub is a git-based code base hosting station, also known as a programmer's SMS (code friend), one of the main open source publishing or hosting station Git is the role of the file version management, convenient for different versions of the switch modification, similar file backup at different times so that when needed to find one of the generations Instead, it is more convenient to use. GitHub is, to a certain extent, a web of code, but also open source developers who publish, store, solicit Bugs and opinions for code.

Why we should learn GIT?

Save time

Git runs fast. Although what we're talking about here is just a few seconds to run a command, accumulating it in your daily routine is a big leap. It can save unnecessary waiting time and accomplish some other meaningful tasks.

Off-line work

How do you work when you can't connect to remote central repository? For a centralized version control system like Subversion or CVS, you can't work well if you don't connect to the central repository. If you use Git, almost everything can be done simply on your local machine. For example, submit, view your project history, merge or create branches and so on. Where do we work? When do you work? Git will not impose any restrictions on you.

Revoking incorrect operation

Everyone makes mistakes, and the greatest benefit of using Git is that you can "undo" your wrong operation in almost all cases. For example, if you forget to include a small change, so you have to correct your last submission. Or you want to cancel a complete submission, because this function may not be necessary. When a serious error occurs, you can even make a commit invisible by restoring the reference log. You can rest assured that Git scarcely really removes data.

High reliability

Don't worry, you won't mess up anything in Git. Is this feeling very good? Every team member on your Git project has cloned the entire project on their local computer, and this local clone can also be considered a complete project backup. In addition, almost all operations on Git are done by adding data, and almost never delete data. This means that it is almost impossible to lose data or warehouse damage.

Making submission more meaningful

Only submission with relevant changes makes sense. Imagine that if a submission includes a newly added feature A, it also includes some changes to function B, and there is a fix for error C. This makes it difficult for other team members to understand the intent of the submission, and when one of the changes goes wrong, it can be cumbersome to undo. With its unique "staging area" concept, Git can help you create subtle and accurate submissions. You can accurately determine which changes will be included in your next submission, even if it's just a one-line change. Git really improves the practicability of version control.

Install GIT and link GIT to GitHub

There are many ways to install Git in Windows. The most formal installer is available for download at Git official website. You only need to connect to http://git-scm.com/download/win and download it automatically. Please note that this is a project called Git for Windows, which is independent from Git itself. If you need more information, please refer to http://git-for-windows.github.io/.

link GIT to GitHub

First thing first, you have to set up your username and email on the git. You can use this command ''git config --global user.name Yourname'' and ''git config --global user.email youre-mail'' to set up your username and email. Then you need a SSH key to make the connection to GitHub. The way to create a SSH key is put this command "ssh-keygen -t rsa -C "youremail"" to your git command line. Then it will create a SSH key according to the email you proving. It will ask you for setting up your password, if you don`t want a password just click enter. After it done, it will show the information underneath.
  1. Your identification has been saved in /c/Users/you/.ssh/id_rsa.
  2. Your public key has been saved in /c/Users/you/.ssh/id_rsa.pub.
  3. The key fingerprint is:
  4. 01:0f:f4:3b:ca:85:d6:17:a1:7d:f0:68:9d:f0:a2:db your email.
When you see that, it means you created a SSH successfully. Then you can use this command "clip < ~/.ssh/id_rsa.pub", to copy your SSH key. Then go to the address here. to link the git with your GitHub account. All you need to do is click add SSH key and paste the key you just copy into the box and click add key. Then you are done. After these steps, you can put '' ssh -T git@github.com'' into your git command line, then put "yes" for the question that comes out, if it shows '' Hi yourGitHubusername! You’ve successfully authenticated, but GitHub does not provide shell access.''. That means the git command line has been connected to your GitHub account.
